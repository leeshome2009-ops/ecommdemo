## 工作规范

### 开发流程
1. 飞书群 @机器人 → 接收需求
2. 理解需求 → 确认理解
3. 调用 Claude Code → 实现代码
4. 运行单元测试 → 确认通过

### 编码规范
- **Java**: 遵循阿里巴巴 Java 开发手册

- **TypeScript**: 严格模式
- **API**: RESTful 风格，统一返回格式 `{ code, message, data }`
- **命名**:

  - Vue: PascalCase (组件), kebab-case (文件)
  - 数据库: snake_case

### 测试规范
- **DevBot**: 只编写单元测试
- **TestBot**: 负责集成测试和联调测试
- **不重叠**: 明确分工，避免重复

## 学习与发现

### Claude-Mem vs OpenViking
- **Claude-Mem**: 记忆压缩系统，专为 Claude Code/OpenClaw 设计
  - 优势: 一键安装，开箱即用，自动化程度高
  - 适用: 快速上手，当前项目

- **OpenViking**: 上下文数据库，火山引擎开源
  - 优势: 文件系统范式，分层加载更精细
  - 适用: 复杂场景，多智能体协作

### Token 优化策略
1. **分层加载**: CONTEXT_SUMMARY.md (700 tokens) vs 完整上下文 (13,000 tokens)
2. **增量更新**: 只传递变更部分
3. **分块存储**: .context/blocks/ 模块化管理
4. **Claude-Mem**: 自动记忆压缩和检索
